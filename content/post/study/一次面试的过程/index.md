## 简述

在 BOSS直聘 上发现的公司，面试的是 Java 相关岗位，时间约到了下午三点钟，上班的地方离家不远，轻轨加电单车约 40 分钟能够抵达，薪资水平在 BOSS 上给出的是 6K，在重庆范围内对应届生来说还不错，也是吸引我去参加面试的因素。

## 过程

首先是拿了一套面试题给我做，题目难度不大，全部都是 Java 的概念基础题，如果好好背了八股文应该来说可以全对，遗憾的是我并没有背八股文，题目靠着自己平时做项目时用的技术和印象答，错了不少。

题型只有单选题和多选题，单选里有考到三元运算符，位运算符，IO 流，多线程，Java 面向对象的概念以及分支条件语句的语法等等，多选题的话涉及了一些抽象类，接口的概念。开发的时候没怎么考虑过这些问题，遇到需求一般都是基于自己的基础然后去 Google ，确实很少巩固自己的基础知识，“基础不牢，地动山摇” 这句话就是今天我做面试题的感受。。

笔试完了过后是面试，面试环节前半段还不错，因为我简历上写的都是自己做的项目，我很熟悉，答起来就比较得心应手。

我先讲了 MyLouise 那个项目，过程中面试官提到：

“我看你这边好像做的都是 Web App 相关，还有后台系统这样的是吗”

我现在想的话，可能是因为这边的公司更需要有实现需求的能力，所以面试官看我做的项目都是后台系统，不太好体现解决问题的能力提出这个问题，但是我当时没考虑到那么多，我以为是后台系统体现不出来开发能力，然后就讲了一下系统设计相关的内容，主要是讲了 MyLouise 如何支持插件系统所做的设计。

然后面试官介绍了一下他的项目组的情况，这会儿我才知道是在做 Android 相关的开发，Android 开发的岗位不好招人，之前已经离职了几位开发人员，而且项目组里面已经有一个新人，已经学习了两个月还不太具备独立开发项目的能力，所以项目组比较缺乏能够立刻上手开发的开发人员。然后面试官就说：

“我们这边可以考虑给你一个机会来加入公司，这样吧，我出一道简单的编程题，找出所有的水仙花数，你给我一个解题思路”

这会儿就坏事了

### 水仙花数

输出所有的“水仙花数”，所谓的“水仙花数”是指一个三位数其各位数字的立方和等于该数本身，例如153是“水仙花数”，因为：153 = 1 + 125 + 27。

我当时一听到水仙花数脑子一懵，因为我曾经是看到过水仙花数的题目，但是除了记得叫水仙花数以外啥都不记得了，所以我先入为主的以为是要给出一种算法意义上的最优解，这导致我一来就把题目都听错，题目限定的是三位数，我后面做的时候一直在尝试计算从 0 到整型类型的最大值的所有水仙花数，直接给我干碎了。。。

等我在错误的思路上爬了十几分钟过后，面试官都打完一个电话之后，我才和他进行第二段沟通，这时我才拿到 “三位数” 这个信息，其实这个时候思路就已经有了，但是我一直认为需要的是一种算法上的最优解，也就一直没敢说最简单的思路。

可能面试官都等麻了，我也急了，然后就用数学意义上的语言描述给面试官解题思路：

“假设这个数是 n，n 除以 100 拿到百位的数字，这个运算过程的余数除以 10 拿到十位的数字，最后剩下的余数就是第三位，将这三个数字的三次方之和相加，如果等于 n 就将它输出”

但是面试官显然对这个回答不是很满意，他需要的是一种更程序的语言，我当时也实在想不出更优秀的解了，就把刚才的数学语言用 Java 再转述了一遍，然后面试官说：

“你的思路是没错的，但是你的代码对于开发来说是比较麻烦，你可能还是对 Java 的一些基础方法还是掌握的不够好，问题在于取每位数字的方法，Java 中提供了将数字转化成数组的方法，或者可以把数字转成字符串，再用 split 方法取得每一位数”

我当时听完愣了，感觉是我没有 Get 到问题的核心是以一种开发更友好的方式完成这个需求，错以为是去寻找数学意义上的最优解，结果一道简单题搞得我心烦意乱。。。

### 尾声

结束面试的时候，面试官让我等面试结果，也没多说其他的。

咋说呢，感觉这次面试在基础的地方确实做得不好，最后水仙花理解上也出了偏差，只能说准备不够充分，如果没面上的话也只有继续努力了，，
